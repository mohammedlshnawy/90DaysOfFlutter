## Why this repo?

I already know how to code, but Flutter and Dart are new to me. So I decided to learn in public — no hiding behind "I'll push when it's clean." Every week there's something here, even if it's rough.

The weekly log is honest. If something confused me, I'll say so. If I skipped a day, that'll show too. The point is to actually finish this, not to make it look like I did.



## The Roadmap

### Phase 1 — Dart Fundamentals (Weeks 1–2)

**Goal:** write a Goal CLI program in Dart by end of week 2

- [x] Types, null safety, variable and functions
- [x] Unit Converter Project
- [x] Classes, and OOP patterns
- [x] Simple Bank Account Project
- [x] Collections: List, Map, Set
- [x] Control Flow
- [x] Grade Tracker Project
- [x] Async / await and Futures
- [x] Fake API Fetcher Project
- [ ] Word Frequency Counter Project


Check Out : [`Phase 1 Notes`](/PhaseOne)

### Phase 2 — Flutter Core (Weeks 3–7)

**Goal:** build a working app that hits a real API by end of week 7

- [ ] Week 3 — Widgets, layout, StatelessWidget
- [ ] Week 4 — StatefulWidget, setState, lifecycle
- [ ] Week 5 — Navigation, routing, multiple screens
- [ ] Week 6 — HTTP requests, JSON, FutureBuilder
- [ ] Week 7 — State management basics (Provider / Riverpod)
- [ ] Weather App



### Phase 3 — Real-World Flutter (Weeks 8–10)

**Goal:** extend the weather app with offline support and polish

- [ ] Week 8 — Local storage: SharedPreferences + SQLite / Isar
- [ ] Week 9 — Forms, validation, user input
- [ ] Week 10 — Custom widgets, themes, basic animations
- [ ] Extend the weather app



### Phase 4 — Capstone Project (Weeks 11–12)

**Goal:** ship one complete, portfolio-ready app from scratch

- [ ] Multi-screen architecture
- [ ] State management throughout
- [ ] Real data (API or local DB)
- [ ] Published on GitHub with its own README
- [ ] Runs on Android and iOS
- [ ] Prayer Times App
